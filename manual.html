03
admin.html — 관리자 화면
📋
명단 탭
슈퍼 전체 마을장
전체 신청자 명단을 카드 형태로 표시합니다. 마을장은 자신의 마을 신청자만 필터링됩니다.
  • 상단 통계 바: 총 인원 / 버스1호차 / 버스2호차 / 자가용 / 참석 현황 표시
  • 검색: 이름·관리번호·목장·마을 실시간 검색
  • 필터: 마을·신청유형·차량·참여일정별 필터
  • 카드 탭: 이름 클릭 → 상세 팝업 (detailPopup 권한 시)
  • 수정: 슈퍼/전체관리자만 정보 수정 가능
  • 삭제: 슈퍼/전체관리자만 행 삭제 가능
📊
통계 탭
슈퍼 전체
마을별·목장별 신청 현황을 차트와 표로 보여줍니다.
  • 마을별 신청 인원 바 차트
  • 신청유형(개인/가족/단체)별 집계
  • 참여일정·차량 유형별 분포
  • 버스 탑승 현황: 1호차 / 2호차 잔여석
🔍
확인필요 탭
슈퍼 전체
주민번호 누락, 전화번호 이상, 동승운전자 미입력 등 데이터 점검이 필요한 건을 자동으로 추출합니다.
  • 주민등록번호 6자리 미입력
  • 전화번호 형식 오류
  • 버스 선택 + 동승운전자 미입력
  • 참여일정·숙소 미입력
💰
회계 탭
슈퍼 전체
납부 현황 관리 및 마을별 정산 요약을 제공합니다.
  • 납부 상태: 미납 / 부분납 / 완납 / 환불
  • 납부일자·금액·메모 직접 입력 후 저장
  • 마을별 납부 집계 테이블
  • 미납자 빠른 필터
  • 납부 데이터는 MET2026납부 시트에 별도 저장
📝
수정로그 탭
슈퍼 전체
누가 언제 어떤 필드를 어떻게 수정했는지 전체 이력을 조회합니다.

  • 본인이 수정한 건: 파란 테두리 + 👤 내 수정 배지 표시
  • 변경 전/후 값 비교 표시

super-admin.html — 슈퍼바이저 전용
🔒 슈퍼바이저(고영수, 전태룡)만 접근 가능합니다. 로그인 시 role이 super가 아니면 입장이 거부됩니다.
👥
사용자 탭
전체 18명의 역할과 상세팝업 권한을 이 화면에서 변경할 수 있습니다.
  • 역할 변경: 슈퍼 / 전체 / 마을장 드롭다운 선택
  • 팝업 권한: 체크박스 토글
  • 변경 건은 노란 점으로 표시 → 하단 "변경사항 저장" 버튼으로 일괄 반영
  • 슈퍼바이저 0명 방지 안전장치 내장
⚠️ 이 화면의 저장은 화면 내 USERS 배열만 변경됩니다. admin.html과 Code.gs에도 별도로 반영해야 완전히 적용됩니다.
📝
수정로그 탭
admin.html의 수정로그보다 강력한 필터링 기능을 제공합니다.
  • 이름 / 관리번호 / 수정자 통합 검색
  • 수정자별 필터 (드롭다운)
  • 변경 필드별 필터 (드롭다운)
  • 요약 표시: 전체 N건 중 M건, 내 수정 K건
  • 본인 수정 건 파란 강조 표시
🗑
삭제이력 탭
수정로그에서 삭제 관련 항목을 자동으로 추출합니다.
  • 현재 GAS에 삭제 로그 미기록 시 → Code.gs 보완 방법 화면에 안내
  • 삭제 로그 기록을 추가하려면 deleteRowsByRno() 함수 안에 logEdit() 호출 추가 필요

구글시트 구조
📋 MET2026신청 (주 데이터)
제출시각 관리번호 그룹번호 신청유형 목장구분 가족관계 이름 식별기호 성별 전화번호 주민등록번호 목자명 마을 목장 사역팀 참여일정 차량 동승운전자 숙소 배우자성명 비고 기기유형

파란색 = 구조값 (수정불가)   초록색 = 수정가능   회색 = 시스템값

💰 MET2026납부
rno pay_status pay_amount pay_date pay_memo updated_at name amount_calc

관리번호(rno) 기준으로 신청 시트와 1:1 연결됩니다.

📝 MET2026수정로그
수정시각 관리번호 이름 수정자 필드 변경전 변경후

수정자 형식: 이름:전화번호4자리 (예: 전태룡:7463). 직접 편집 금지.

🔢 _카운터

A1: 개인(A) 카운터 · B1: 가족(F) 카운터 · C1: 단체(G) 카운터
관리번호 자동 부여 시 이 값을 읽고 +1 저장합니다. 직접 수정하면 번호 중복이 발생할 수 있으니 반드시 GAS유틸 재정렬을 사용하세요.

08
자주 묻는 질문 (FAQ)
Q새 관리자를 추가하려면?
admin.html, super-admin.html, Code.gs 세 곳의 USERS 배열에 동일하게 추가합니다.
Code.gs 수정 후 재배포 필수. 형식:
{ name:'홍길동', phone:'1234', role:'admin', detailPopup:false }
Q관리번호가 중복됐거나 순서가 엉켰다면?
super-admin.html → GAS유틸 탭 → 관리번호 재정렬을 실행합니다.
제출시각 기준으로 A/F/G 번호를 전체 재부여하며 _카운터 시트도 자동 갱신됩니다.
Q본인이 수정한 건도 수정로그에 기록되나요?
네, 예외 없이 기록됩니다. 실제로 값이 바뀐 필드만 기록되며, 수정 창을 열었다 그냥 닫으면 로그가 남지 않습니다.
본인 수정 건은 수정로그 화면에서 파란 강조 + 👤 내 수정 배지로 구분됩니다.
Q버스 잔여석은 어디서 확인하나요?
admin.html 명단 탭 상단 통계 바에서 실시간 확인 가능합니다.
버스 1호차 / 2호차 각각 39석 기준. 슈퍼바이저는 GAS유틸 탭 → 버스 현황 조회를 이용할 수 있습니다.
Q삭제한 신청자 기록을 복구하려면?
현재 삭제 시 수정로그에 기록되지 않습니다. 구글시트 MET2026신청의 버전 기록(파일 → 버전 기록)을 통해 이전 상태로 복원할 수 있습니다.
향후 삭제 이력 자동 기록 기능 추가 예정.
QGAS 유틸 실행 후 "처리 실패"가 뜬다면?
해당 유틸 액션이 Code.gs에 구현되어 있지 않은 경우입니다.
슈퍼바이저에게 연락하여 Code.gs에 해당 핸들러를 추가하고 재배포를 요청하세요.
Q이메일 알림이 오지 않는다면?
Apps Script 편집기에서 testEmail() 함수를 직접 실행하거나, super-admin.html GAS유틸 탭 → 테스트 이메일 발송을 실행합니다.
GAS의 하루 이메일 발송 한도(100건)를 초과했는지도 확인하세요.
